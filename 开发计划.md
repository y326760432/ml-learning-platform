## 📝 附录

### A. Git分支策略

```
main          ←─ 生产环境
  ↑
  merge
  ↑
develop       ←─ 开发主分支
  ↑
  merge
  ↑
feature/*     ←─ 功能分支
bugfix/*      ←─ Bug修复分支
hotfix/*      ←─ 紧急修复分支
```

### B. API设计规范

#### RESTful规则
- GET: 获取资源
- POST: 创建资源
- PUT: 更新资源（完整）
- PATCH: 更新资源（部分）
- DELETE: 删除资源

#### 示例
```
GET    /api/courses              获取课程列表
GET    /api/courses/:id          获取课程详情
POST   /api/courses              创建课程
PUT    /api/courses/:id          更新课程
DELETE /api/courses/:id          删除课程

GET    /api/courses/:id/lessons  获取课程章节
POST   /api/progress             更新学习进度
```

### C. 数据库备份策略
- 每日全量备份（凌晨3点）
- 每小时增量备份
- 保留最近7天的备份
- 每月归档备份

### D. 监控指标
- 服务可用性
- API响应时间
- 错误率
- 用户活跃度
- 代码执行次数
- 服务器资源使用率
