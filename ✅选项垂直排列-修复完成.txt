🎯 问题描述
═══════════════════════════════════════════════════════════════

  用户反馈：
  "选项还水平排列的"


  问题分析：
  - 按钮元素默认 display: inline-block
  - 导致多个按钮尝试在同一行显示
  - 虽然设置了 w-full 和 space-y-3
  - 但 inline-block 仍然会尝试横向排列


🔧 解决方案
═══════════════════════════════════════════════════════════════

  修改 src/components/QuizQuestion.tsx

  关键代码改动：
  ```typescript
  <button
    className={`block w-full text-left ...`}
    //        ^^^^^ 添加 block
  >
  ```

  说明：
  - 添加 `block` 到 className
  - 确保按钮独占一行
  - 配合 space-y-3 实现垂直间距


📝 技术细节
═══════════════════════════════════════════════════════════════

  CSS Display 属性：

  ❌ inline-block
     - 元素会尝试在同一行显示
     - 即使设置了 width: 100%
     - 仍可能并排显示

  ✅ block
     - 元素独占一行
     - 自动换行
     - 垂直堆叠


  完整 className：
  block w-full text-left p-4 rounded-xl border-2 
  transition-all duration-300 ...

🔍 学到的教训
═══════════════════════════════════════════════════════════════

  1. inline-block 的陷阱
     - 即使设置 width: 100%
     - 元素仍可能尝试并排显示
     - 需要显式设置 display: block

  2. Tailwind CSS 的 block 类
     - 显式声明布局模式
     - 避免浏览器默认行为
     - 确保预期的布局

  3. 测试的重要性
     - 代码中看起来正确（space-y-3）
     - 但实际渲染可能不同
     - 需要在浏览器中验证
