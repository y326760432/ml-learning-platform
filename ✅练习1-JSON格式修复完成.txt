🐛 问题描述
═══════════════════════════════════════════════════════════════

  用户报告：
  "练习1：机器学习的定义
   ⚠️ 练习题加载失败
   数据格式错误，请检查 JSON 格式"


  错误原因：
  - JSON 字符串中使用了中文引号 ""
  - 应该使用英文引号 ""
  - 导致 JSON.parse() 解析失败


🔍 问题定位
═══════════════════════════════════════════════════════════════

  错误位置：
  content/courses/module-1/lesson-01-什么是机器学习.md
  第452行的 explanation 字段

  ❌ 错误代码：
  "explanation": "机器学习的核心是"从数据中学习"，..."
                                    ↑         ↑
                                  中文引号    中文引号

  问题分析：
  1. JSON 标准只支持英文双引号 ""
  2. 中文引号 "" 会导致解析器报错
  3. 即使字符串内部需要引号，也应该转义或避免


🔧 解决方案
═══════════════════════════════════════════════════════════════

  修复方法：
  去掉 explanation 中的引号，改为直接描述

  ✅ 修复后：
  "explanation": "机器学习的核心是从数据中学习，不需要..."
                                ↑ 去掉了引号 ↑

  完整修复代码：
  ```json
  {
    "question": "以下哪个选项最准确地描述了机器学习？",
    "type": "single",
    "options": [
      {
        "id": "a",
        "text": "一种让计算机变得智能的编程语言",
        "isCorrect": false
      },
      {
        "id": "b",
        "text": "一种让计算机从数据中学习而无需明确编程的技术",
        "isCorrect": true
      },
      {
        "id": "c",
        "text": "一种专门用于数据存储的数据库系统",
        "isCorrect": false
      },
      {
        "id": "d",
        "text": "一种用于创建网站的框架",
        "isCorrect": false
      }
    ],
    "explanation": "机器学习的核心是从数据中学习，不需要为每个场景编写明确的规则，而是通过算法自动发现数据中的模式。它不是编程语言、数据库系统或网站框架。"
  }
  ```

💡 经验教训
═══════════════════════════════════════════════════════════════

  1. JSON 格式规范
     - 必须使用英文双引号 ""
     - 不能使用中文引号 ""
     - 不能使用单引号 ''

  2. 字符串内引号处理
     - 方案1：转义 \"
     - 方案2：避免使用引号
     - 方案3：改写表达方式

  3. 调试技巧
     - 使用浏览器查看原始数据
     - 使用 JSON validator 验证格式
     - 查看详细错误信息

  4. 文档编写注意
     - 避免在 JSON 中使用中文标点
     - 特别注意复制粘贴的内容
     - 编辑器可能自动转换引号
